Concepts used in the RealEstateCore ontology described briefly.

Building structure

The highest level in the hierarchy is RealEstate that consist of BuildingStructure(s) and Land(s). RealEstate is the legal entity.

A BuildingStructure consists of one or more BuildingStructureComponents.

A BuildingStructureComponent can be: Room, Roof, Facade, VirtualComponent, etc.

A VirtualComponent is a BuildingStructureComponent that not need to physically exist but is useful to attach data that are calculated or received from external sources (e.g. an energy meter reading from a utility company).

Premises = a collection of Rooms or other spaces or objects that are leased to a tenant.

StoreyLevel = a horizontally aligned collection of BuildingStructureComponents. Can be a popular name or a numbering.

Location

Location is used to describe an object's or event's geographical placement.

locatedIn
An object that can describe its place by using [myObject-UUID] locatedIn [myOtherObject-UUID], where [myOtherObject-UUID] could be a Room, BuildingStructure, etc.

locatedAt
An object that has an exact point on Earth (expressed in WGS84).
E.g. [myObject-UUID] locatedAt [lat;long;alt].

Device

A Device is a thing made or adapted for a particular purpose, specifically a piece of electronic equipment.

A Device has one or more Sensor(s) and/or Actuator(s) and/or other Device(s).

A Sensor is a thing which detects or measures a physical property and records, indicates, or otherwise responds to it.

An Actuator is a component that is responsible for moving and controlling a mechanism or system.

Observation

An Observation is made by a Sensor and hasValue and hasObservationTime and often also hasMeasurmentUnit.

QuantityKind

Describes the value observed by a Sensor or the value used by an Actuator.

Core has a list of preset instances of different types of QuantityKinds (e.g. Power, Flow or Temperature) to define a controlled vocabulary.

PlacementContext

In what context or media that the Sensor or Actuator is operating in. E.g in the SupplyAir or the ExhaustAir flow.

Core has a list of preset instances of different types of PlacementContext (e.g. AirIntake, DistrictHeatingFlow or DistrictHeatingReturn) to define a controlled vocabulary.
